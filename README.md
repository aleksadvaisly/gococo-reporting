# GoCoCo Reporting

<p align="center"><img src="docs/goreport.png" width="50%"></p>

`gococo-reporting` is a small Go library and CLI for generating a single self-contained HTML report for BDD-style test results.

The output is designed to be easy to share and easy to open offline:

- one `.html` file
- inline CSS and JavaScript
- no external assets
- rounded cards and grid-based layout
- soft pastel color palette

## What It Supports

Current v1 input formats:

1. Cucumber JSON arrays, such as output generated by:

```bash
godog --format=cucumber:report.json
```

2. `go test -json` streams produced by nested BDD-style tests, including `gobdd`-style test hierarchies.

That makes the tool useful both for direct cucumber-compatible JSON and for Go-native BDD test runs.

## Why This Project Exists

The goal is a compact reporting tool with a very small footprint:

- simple Go code
- no unnecessary object-heavy design
- one-file report output
- usable both as a library and as a command-line tool

## Build

```bash
make
make build
```

The binary is created at:

```text
bin/gococo-reporting
```

## CLI Usage

Single input file:

```bash
./bin/gococo-reporting \
  -input report.json \
  -output report.html \
  -title "My BDD Report"
```

Multiple input files:

```bash
./bin/gococo-reporting \
  -input part1.json,part2.json \
  -output report.html \
  -title "Merged Report"
```

You can also pass positional files after flags:

```bash
./bin/gococo-reporting \
  -output report.html \
  report-1.json report-2.json
```

## Make Targets

```bash
make
```

Available targets:

- `build` builds the CLI into `bin/`
- `test` runs all Go tests
- `fmt` formats Go files
- `tidy` runs `go mod tidy`
- `clean` removes build artifacts
- `run` runs the CLI with `INPUT=... OUTPUT=... TITLE=...`

## Library Usage

```go
package main

import (
	"log"
	"time"

	"github.com/aleksadvaisly/gococo-reporting"
)

func main() {
	err := gococo.WriteHTMLReport(
		"report.html",
		[]string{"report.json"},
		gococo.Options{
			Title:       "Acceptance Report",
			GeneratedAt: time.Now(),
		},
	)
	if err != nil {
		log.Fatal(err)
	}
}
```

Public API:

- `GenerateHTMLFromFiles`
- `WriteHTML`
- `WriteHTMLReport`

## Output Characteristics

The generated HTML includes:

- top-level summary
- feature cards
- collapsible scenarios
- step details
- status filtering
- text search
- failure section for failed steps

The report is fully static after generation. No server is required to view it.

## Status

This is currently a focused v1.

Intentionally out of scope for now:

- multi-page HTML output
- external charting libraries
- heavyweight configuration layers
- broad compatibility with every cucumber ecosystem variant

The current priority is keeping the tool small, understandable, and useful.
